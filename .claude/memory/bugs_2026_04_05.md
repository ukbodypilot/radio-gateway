## Session Summary — 2026-04-05

Major bug-fix session touching bus processing, SDR noise, endpoint mode switching, and routing UI.

### Critical architectural lesson: IIR filters are stateful
AudioProcessor uses IIR filters (HPF/LPF/notch). Calling `.process()` advances internal filter state. If called N times per tick (once per sink), the filter output is wrong for all but the first call. **Process audio ONCE per bus tick, then distribute to sinks.**

### SDR continuous mode
rtl_airband `continuous = true` sends noise floor to PipeWire when squelch is closed. This polluted routing levels (constant ~71 on SDR/bus). Fix: hardcode `continuous = false` in config generation. StreamOutputSource silence keepalive handles Broadcastify independently. A noise gate approach was attempted first but abandoned — noise floor and weak signals overlap in dB, can't distinguish them in the audio domain.

### Endpoint mode switch
Two bugs: (1) `_send_endpoint_mode()` silently failed → endpoint stuck in data mode, (2) PyAudio instance not terminated → ALSA device busy for Direwolf. Both fixed. Endpoint status now visible on /packet UI with Force Audio recovery button.

### Web UI nav
Shell nav bar reorganized into dropdown groups: Audio, Radios, Data, System. Pure CSS/JS, click-to-open dropdowns.
