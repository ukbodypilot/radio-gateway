## 2026-04-01 Bug Log

**TH-9800 PortAudio segfault (3 overnight crashes):** False watchdog triggered by `gateway.last_audio_capture_time` only updating on primary listen bus. Reader thread refactored to own stream lifecycle entirely.

**Stale `self.input_stream`:** Gateway cached plugin's stream at init. Reader thread close/reopen made it stale → `OSError: Stream not open` on every main loop tick.

**`VAD_ENABLED` vs `ENABLE_VAD`:** Wrong config attribute name silently crashed main loop on every tick, preventing ALL sink delivery.

**TH-9800 audio glitches:** `read_size = chunk_size * 4` (9600) exceeded ALSA buffer (4800) causing overruns. Fixed: read one period (2400) with larger buffer (`frames_per_buffer=chunk*4`). Confirmed clean by sample-level analysis (boundary ratio 1.0x).

**Mumble frame alignment stutter:** 50ms chunks (2400 samples) don't divide evenly into 20ms Mumble frames (960 samples). Fractional accumulation caused progressive stutter after ~10s. Fixed: buffer and feed exact 20ms frames.

**PCM WebSocket clock drift:** Main loop and BusManager run separate 50ms clocks. Main loop mixed both sources' PCM causing intermittent gaps. Fixed: BusManager pushes its own PCM directly to WebSocket.

**Python 3.14 threading scoping:** Local `import threading` in web_server.py handler methods shadowed module-level import → `UnboundLocalError` breaking WebSocket connections.

**Cloudflare tunnel:** `KillMode=control-group` killed cloudflared. Fixed with `systemd-run --user --scope`. Removed HEAD validation (backend not up during startup).

**D75 link endpoint issues:** Queue competition (primary mixer + solo bus both calling get_audio), stale `_get_plugin_by_id` returning None, missing `execute/put_audio` on LinkAudioSource, FO readback not updating cached state, `to_dict` missing template merge, field name mismatches (s_meter→signal, memory_mode int→string).

**Routing page position collision:** Source "announce" and bus "announce" shared savedPositions key.
