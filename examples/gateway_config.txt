# ============================================================================
# RADIO GATEWAY - CONFIGURATION FILE
# ============================================================================
#
# Bidirectional audio gateway between Mumble VoIP and radio via AIOC
#
# Quick Start:
#   1. Set MUMBLE_SERVER to your server IP/hostname
#   2. Most other settings have sensible defaults
#   3. Optional features are disabled by default — enable as needed
#
# Keyboard Controls (runtime):
#   MUTE:  't'=TX  'r'=RX  'm'=Global  's'=SDR1  'x'=SDR2  'c'=Remote  'a'=Announce  'o'=Speaker
#   AUDIO: 'v'=VAD toggle  ','=Vol-  '.'=Vol+
#   PROC:  'n'=Gate  'f'=HPF  'g'=AGC  'y'=Spectral  'w'=Wiener  'e'=Echo
#   SDR:   'd'=SDR1 Duck toggle  'b'=SDR Rebroadcast toggle
#   PTT:   'p'=Manual PTT toggle
#   PLAY:  '1-9'=Announcements  '0'=Station ID  '-'=Stop
#   NET:   'k'=Reset remote audio connection
#   RELAY: 'j'=Radio power button  'h'=Charger toggle
#   SMART: '['=Smart#1  ']'=Smart#2  '\'=Smart#3
#   TRACE: 'i'=Start/stop audio trace  'u'=Start/stop watchdog trace
#   MISC:  'q'=Restart gateway  'z'=Clear and reprint console
# ============================================================================

# ============================================================================
# SIGNAL DETECTION & SOURCE SWITCHING
# ============================================================================
# Controls ducking transitions: when radio RX arrives and ducks SDR, and
# when it ends and SDR resumes.
#
# Recommended starting values:
#   Sensitive/fast:   attack=0.1,  release=1.0, padding=0.5
#   Balanced:         attack=0.25, release=3.0, padding=1.0
#   Conservative:     attack=0.5,  release=5.0, padding=1.0

# Seconds of CONTINUOUS signal before a duck is triggered

# Seconds of continuous silence before the ducked source resumes

# Seconds of silence inserted at each transition (duck-out AND duck-in)

# SDR-to-SDR: seconds of immunity after unduck before re-duck is allowed
# Prevents rapid toggling with intermittent signals. 0=no cooldown.

# SDR signal threshold (dBFS). Chunks below this are treated as silence.
# -50=default | -60=10dB more sensitive (good for weak signals)

# SDR Rebroadcast: seconds to hold PTT after SDR audio signal stops.

# ============================================================================
# TROUBLESHOOTING
# ============================================================================
#
# AUDIO:
#   No audio on Mumble     -> Check INPUT_VOLUME, VAD/VOX settings
#   No audio on radio      -> Check OUTPUT_VOLUME, AIOC connection
#   Choppy / -9999 errors  -> Increase AUDIO_CHUNK_SIZE
#
# MUMBLE:
#   Won't connect          -> Verify MUMBLE_SERVER and MUMBLE_PORT
#   Channel not found      -> Check MUMBLE_CHANNEL spelling
#
# PTT:
#   Doesn't activate       -> Try AIOC_PTT_CHANNEL = 1, 2, or 3
#   Cuts off too fast      -> Increase PTT_RELEASE_DELAY
#
# SDR:
#   Goes silent after hours -> Watchdog will auto-recover (see SDR_WATCHDOG_*)
#   Status shows W1, W2    -> Recovery attempts made; !restart if still silent
#
# STREAMING:
#   Stream offline          -> Check DarkIce is running (ps aux | grep darkice)
#   No audio on stream      -> Verify FFmpeg bridge is running
#
# For more help, enable VERBOSE_LOGGING = true
# ============================================================================
